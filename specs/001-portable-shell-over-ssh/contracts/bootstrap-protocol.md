# Contract: Host Bootstrap Protocol

**Crate**: `xxh-core` + `bootstrap/bootstrap.sh` | **Principles**: I, V, VI

Порядок разворачивания окружения на хосте по установленному SSH-соединению. Минимальный
контракт хоста: POSIX `sh` + базовые утилиты (cat, mkdir, chmod, tar/gzip). Root не нужен.

## Последовательность

1. **Detect platform**: `exec` → `uname -s -m` (+ `command -v zstd gzip tar`).
   Неподдерживаемая платформа → `ShellError::Unsupported`, стоп до любой записи на хост
   (FR-007).
2. **Reconcile**: `exec` листинг `~/.xxh/sessions` и `~/.xxh/cache`. Устаревшие
   session-маркеры (без живого процесса) → очистить остатки прошлых аварийных сессий
   (FR-006, edge «разрыв соединения»).
3. **Diff cache**: клиент получает set имеющихся `<blake3>` из `~/.xxh/cache`; вычисляет,
   какие Component отсутствуют (FR-013, Принцип VI).
4. **Deliver**: только отсутствующие компоненты передаются `upload_stream` (tar+zstd, или
   tar+gz если zstd нет) в bootstrap, который распаковывает в `~/.xxh/cache/<blake3>/`.
   Нет места/прерывание → откат частично записанного, хост чист (FR-032).
5. **Assemble**: bootstrap линкует/раскладывает компоненты в рабочую директорию сессии,
   подключает конфиги, PATH, рантайм-данные (сертификаты/terminfo/локали ⭐).
6. **Hooks post_deploy**: запуск объявленных хуков как отдельных процессов с таймаутом;
   сбой хука → изолированная `PluginError`, сессия продолжается (FR-019).
7. **Launch shell**: интерактивный PTY с выбранным шеллом (FR-002).
8. **pre_exit hooks + Teardown**: при выходе — хуки `pre_exit`, затем очистка.

## Cleanup-контракт (Принципы I, V)

- Bootstrap устанавливает `trap 'xxh_cleanup' EXIT INT TERM HUP`.
- `Ephemeral` (дефолт): `xxh_cleanup` удаляет `~/.xxh` (включая кеш) → хост чист (FR-005,
  SC-002).
- `Keep` (`--keep`): сохраняется `~/.xxh/cache` для ускорения повторного входа; session-
  директория и маркер удаляются (FR-012).
- Если `trap` не сработал (kill -9 / обрыв) → шаг 2 (Reconcile) следующей сессии вычищает
  остатки (FR-006, SC-007).

## Обязательства

- **C-B1**: Ни один artefact не пишется на хост до успешного Detect (FR-007).
- **C-B2**: Всё — только через SSH-сессию (FR-030).
- **C-B3**: Скрипт не содержит и не логирует секретов (Принцип V).
- **C-B4**: Идемпотентность по `<blake3>`: повторная доставка того же компонента —
  no-op (Принцип VI).

## Тестируемость (smoke, merge-gate)

Сценарий «подключились → получили свой шелл → вышли → на хосте чисто»:
проверка `~/.xxh` отсутствует после Ephemeral-сессии; проверка ускорения при `--keep`;
проверка очистки остатков после симулированного обрыва (`cleanup_crash.rs`).
