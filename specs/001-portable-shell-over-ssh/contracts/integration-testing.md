# Contract: Integration Testing Infrastructure

**Files**: `tests/integration/*`, `tests/images/*.Dockerfile`, `.github/workflows/integration.yml`
| **Principle**: VIII (Тестируемость), синергия I, II, VI, IX, X

Интеграция против **реальных `sshd`-контейнеров** по матрице libc/coreutils. Ни один
сценарий с ожидаемой очисткой не считается пройденным без **ассерта чистоты хоста**.

## Тестовые образы (`tests/images/`)

| Образ | libc / coreutils | Роль |
|-------|------------------|------|
| `debian.Dockerfile` | glibc + GNU | базовый glibc-кейс |
| `ubuntu.Dockerfile` | glibc + GNU | второй glibc-кейс |
| `alpine.Dockerfile` | musl + BusyBox | **критичный** кейс (POSIX sh, урезанный uname) |

- **C-IT1**: Образ содержит только `sshd` + непривилегированную тестовую учётку; никаких
  предустановленных шеллов-плагинов/инструментов/интерпретаторов сверх базового образа.
- **C-IT2**: Учётка НЕ имеет прав ставить системные пакеты (нет sudo/root) в рамках
  сценария — честная проверка zero-footprint и работы без предустановленных зависимостей
  (Принципы I, II).
- **C-IT3**: Аутентификация по ключу (пара генерируется в тесте); `sshd` с известным
  портом и **фиксированным host key** для стабильного `known_hosts`.
- **C-IT4**: Alpine обязателен — bootstrap и platform-detection ДОЛЖНЫ корректно работать
  на BusyBox/musl (урезанный `uname`/coreutils, нет glibc).

## Оркестрация

- **C-IT5**: Основной способ подъёма — `testcontainers-rs` (из Rust, RAII/Drop-teardown) —
  CI-детерминизм и гарантированная очистка даже при падении теста. `docker-compose`
  допустим только для локального ручного прогона.
- **C-IT6**: Каждый обязательный сценарий выполняется против **всех трёх** дистрибутивов;
  где релевантно — против нескольких арх (aarch64 через qemu/binfmt), с общей target-
  таблицей Nix-провайдера (nix-provider.md, research R11).

## Обязательные сценарии и ассерты

| ID | Сценарий | Ассерт |
|----|----------|--------|
| C-IT-S1 | connect → свой шелл → интерактивная команда → выход | prompt/алиас доступен; команда отработала (§FR-002) |
| C-IT-S2 | чистота хоста после выхода | `~/.xxh` и артефакты удалены — отдельный ssh-заход/проверка перед teardown (§FR-005, §SC-002) |
| C-IT-S3 | аварийный разрыв посреди сессии | при следующем заходе окружение вычищается (§FR-006, §SC-007) |
| C-IT-S4 | повторное подключение | кеш переиспользован, уже развёрнутое не перезалито (§FR-013, §SC-004) |
| C-IT-S5 | флаг сохранения окружения | артефакты остаются между сессиями при явном флаге (§FR-012) |
| C-IT-S6 | плагин из git/local | плагин применён; сбой одного плагина не роняет сессию (§FR-016/019, §SC-006) |
| C-IT-S7 | ⭐ (feature `nix-source`) nix-статик на Alpine | артефакт доставлен и запущен без Nix на хосте (§FR-034/035, §SC-010) |

- **C-IT7** (правило Принципа VIII): сценарий «сессия открылась» без ассерта чистоты —
  **не пройден**. Ассерт чистоты обязателен во всех сценариях с ожидаемой очисткой
  (S1, S3; для S5 — обратный ассерт: артефакты присутствуют).

## Интеграция с Nix и путь без Nix

- **C-IT8**: Образы и запуск описываются воспроизводимо; `flake checks` содержит job
  интеграции там, где доступны docker/qemu.
- **C-IT9** (anti-lock-in): те же тесты запускаются обычным `cargo test` с внешним docker,
  без Nix (Принцип X); оркестрация одинакова.

## CI (`integration.yml`)

- **C-IT10**: Отдельный workflow; матрица `{Debian, Ubuntu, Alpine} × {x86_64, aarch64}`.
- **C-IT11**: Кеширование образов между прогонами.
- **C-IT12**: Таймаут на сценарий и **гарантированный teardown** контейнеров даже при
  падении (Drop `testcontainers-rs` + cleanup-step workflow).
- **C-IT13**: Прохождение всей обязательной матрицы (glibc+GNU и musl+BusyBox) с ассертами
  чистоты — merge-gate (Принцип VIII, quality-gate 2).
