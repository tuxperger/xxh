# CLI & Configuration Requirements Quality Checklist: Portable Shell Environment over SSH

**Purpose**: Проверка КАЧЕСТВА требований к CLI и конфигурации (полнота, однозначность,
непротиворечивость, измеримость) — interrogate требования, не реализацию. Статус:
**[Покрыто]** — задано и однозначно, **[Двусмысленно]** — есть, но недоопределено,
**[Пробел]** — отсутствует.
**Created**: 2026-07-03
**Feature**: [spec.md](../spec.md) (US5, US6, FR-022..028) · сверено с
[contracts/cli-commands.md](../contracts/cli-commands.md) · Принцип VII

## Полнота набора подкоманд и аргументов

- [ ] CHK001 Определён ли **полный набор подкоманд** (connect, plugin add/remove/enable/disable/update/list, config)? [Completeness, Покрыто, Spec §FR-015 / Contract cli-commands]
- [ ] CHK002 Специфицированы ли **аргументы/флаги** подключения (`--shell`, `--keep`, `--transport`, `--connect-timeout`, `-v/-vv/--debug`) без двусмысленности? [Clarity, Покрыто, Contract cli-commands (таблица опций)]
- [ ] CHK003 Поднят ли набор подкоманд в **нормативный уровень spec** (§FR), а не только contract? [Traceability, Двусмысленно, детальная таблица команд — в contract; §FR-015 задаёт лишь операции плагинов]
- [ ] CHK004 Определены ли **обязательные vs опциональные** аргументы каждой подкоманды? [Clarity, Двусмысленно, Contract перечисляет опции, обязательность (кроме `<host>`) не размечена]
- [ ] CHK005 Задан ли **формат source-аргумента** для `plugin add` (git-url / path / ⭐nixpkgs:attr) однозначно? [Clarity, Покрыто, Contract cli-commands / Spec §FR-016/033]
- [ ] CHK006 Определено ли поведение при **неизвестной подкоманде/аргументе** (справка, код выхода)? [Edge Case, Двусмысленно, Contract `--help` есть; реакция на невалидный ввод не специфицирована]

## Приоритеты конфиг / флаги / пер-хостовые оверрайды

- [ ] CHK007 Определён ли **порядок приоритета** (флаг > пер-хост > глобальный конфиг > дефолт)? [Completeness, Покрыто, Spec §FR-024 / Clarifications 2026-07-03]
- [ ] CHK008 Задано ли применение **пер-хостовых переопределений** поверх глобальных? [Completeness, Покрыто, Spec §FR-023]
- [ ] CHK009 **Непротиворечивы** ли требования приоритета между §FR-023 (пер-хост) и §FR-024 (флаги) — нет ли конфликта? [Consistency, Покрыто, Spec §FR-023/§FR-024]
- [ ] CHK010 Однозначно ли, как **сливаются** списочные значения (напр. enabled_plugins) при оверрайде — замена или объединение? [Clarity, Пробел, Gap]
- [ ] CHK011 Определён ли способ **просмотра эффективной конфигурации** с учётом приоритетов? [Coverage, Покрыто, Contract cli-commands (`config show`)]

## Формат и поля конфиг-файла

- [ ] CHK012 Специфицирован ли **формат** конфиг-файла и его расположение? [Completeness, Двусмысленно, data-model/Contract задают TOML+путь; §FR-022 формат не называет]
- [ ] CHK013 Определён ли **полный набор полей** конфига (default_shell, enabled_plugins, cleanup, transport, connect_timeout, hosts)? [Completeness, Покрыто, Spec §FR-022 / data-model Config]
- [ ] CHK014 Заданы ли **значения по умолчанию** для каждого поля? [Completeness, Покрыто, data-model Config (таблица дефолтов)]
- [ ] CHK015 Определена ли **обязательность** каждого поля (что MUST присутствовать, что опционально)? [Clarity, Двусмысленно, data-model задаёт дефолты (⇒ опциональность), но нормативно в §FR не зафиксировано]
- [ ] CHK016 Заданы ли **допустимые значения/валидация** полей (напр. transport ∈ {russh, ssh})? [Clarity, Двусмысленно, data-model перечисляет enum-значения; правила валидации ввода не заданы]
- [ ] CHK017 Определена ли **схема пер-хостового блока** (`hosts.<alias>`) и какие поля в нём допустимы? [Completeness, Покрыто, data-model HostOverride]

## Режимы вывода (обычный / -v/--verbose / debug)

- [ ] CHK018 Определены ли **режимы вывода** (обычный, `-v/--verbose`, debug)? [Completeness, Покрыто, Spec §FR-027 / §FR-025]
- [ ] CHK019 Специфицировано ли, **что именно показывает** каждый уровень детализации (разграничение normal/-v/-vv/debug)? [Clarity, Двусмысленно, Contract упоминает уровни; содержимое каждого уровня не детализировано]
- [ ] CHK020 Задано ли требование **прогресса этапов** подготовки как части нормального вывода? [Coverage, Покрыто, Spec §FR-025]
- [ ] CHK021 Непротиворечив ли verbose-вывод с **запретом логирования секретов** во всех режимах? [Consistency, Покрыто, Spec §FR-028 / §FR-027]

## Различимость классов ошибок для пользователя

- [ ] CHK022 Различимы ли в требованиях **классы ошибок** (транспорт / сборка шелла / плагин)? [Completeness, Покрыто, Spec §FR-026]
- [ ] CHK023 Заданы ли **коды выхода** по классам ошибок однозначно? [Clarity, Двусмысленно, Contract cli-commands (0/10/20/30/40) есть; в §FR коды не зафиксированы]
- [ ] CHK024 Покрыт ли класс **config-ошибок** отдельно от транспорт/шелл/плагин? [Coverage, Двусмысленно, Contract вводит код 40 (config); §FR-026 называет только три класса]
- [ ] CHK025 **Измеримо** ли, что пользователь определяет класс ошибки без внешней документации? [Measurability, Покрыто, Spec §SC-005]

## Поведение при некорректном / отсутствующем конфиге

- [ ] CHK026 Задано ли поведение при **отсутствующем конфиге** (использовать дефолты)? [Edge Case, Двусмысленно, data-model задаёт дефолты (подразумевает работу без конфига), явного §FR нет]
- [ ] CHK027 Определено ли поведение при **некорректном/непарсящемся конфиге** (ошибка класса config, не падение)? [Edge Case, Пробел, Gap]
- [ ] CHK028 Задано ли поведение при **неизвестных/лишних полях** конфига (игнор/предупреждение/ошибка)? [Edge Case, Пробел, Gap]
- [ ] CHK029 Определено ли поведение при **ссылке на несуществующий плагин/шелл** в конфиге? [Edge Case, Двусмысленно, §FR-011 покрывает отсутствующий шелл; отсутствующий плагин в enabled_plugins — не задан]

## Совместимость указания хоста с ~/.ssh/config

- [ ] CHK030 Задано ли, что **имя хоста совместимо с записями `~/.ssh/config`** так, как ожидает пользователь? [Completeness, Покрыто, Spec §FR-001 / §FR-029]
- [ ] CHK031 Определено ли, какие **параметры ssh-config** учитываются (HostName, User, Port, ProxyJump и т.д.)? [Clarity, Двусмысленно, §FR-029 называет ProxyJump/agent/known_hosts; полный перечень учитываемых директив не зафиксирован]
- [ ] CHK032 Непротиворечивы ли **пер-хостовые оверрайды xxh** и параметры из ssh-config (кто за что отвечает, нет ли конфликта семантики «host»)? [Conflict, Двусмысленно, §FR-023 (xxh hosts) и §FR-029 (ssh config) не разграничены явно]
- [ ] CHK033 Определено ли поведение, если алиас **есть в xxh-конфиге, но отсутствует в ssh-config** (или наоборот)? [Edge Case, Пробел, Gap]

## Измеримость качества сообщений об ошибках (actionability)

- [ ] CHK034 **Измеримы** ли требования к полезности сообщений об ошибках (actionability — подсказка следующего шага)? [Measurability, Двусмысленно, Spec §FR-011 требует «указания как добавить»; общий критерий actionability для всех ошибок не задан]
- [ ] CHK035 Задан ли **минимальный стандарт** сообщения об ошибке (класс + причина + что делать)? [Completeness, Пробел, Gap]
- [ ] CHK036 Определена ли **проверяемость** «понятности» ошибок (SC как критерий приёмки)? [Measurability, Покрыто, Spec §SC-005]

## Трассируемость и открытые вопросы

- [ ] CHK037 Подняты ли CLI/config-требования до **нормативного §FR** (а не только contract/data-model)? [Traceability, Двусмысленно, формат конфига, коды выхода, таблица команд живут в contract/data-model]
- [ ] CHK038 Занесены ли все **[Двусмысленно]/[Пробел]** (CHK003, 004, 006, 010, 012, 015, 016, 019, 023, 024, 026, 027, 028, 029, 031, 032, 033, 034, 035, 037) как открытые вопросы? [Ambiguity, Action]
