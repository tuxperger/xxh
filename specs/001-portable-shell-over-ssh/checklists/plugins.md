# Plugin System Requirements Quality Checklist: Portable Shell Environment over SSH

**Purpose**: Проверка КАЧЕСТВА требований к системе плагинов (полнота, однозначность,
непротиворечивость, измеримость) — interrogate требования, а не код. Статус каждого пункта:
**[Покрыто]** — задано и однозначно, **[Двусмысленно]** — есть, но неоднозначно/недоопределено,
**[Пробел]** — отсутствует в требованиях.
**Created**: 2026-07-03
**Feature**: [spec.md](../spec.md) · сверено с [contracts/plugin-manifest.md](../contracts/plugin-manifest.md), [contracts/plugin-source-trait.md](../contracts/plugin-source-trait.md), [contracts/nix-provider.md](../contracts/nix-provider.md) · Принцип IV, IX

## Операции жизненного цикла (install/enable/disable/update/remove)

- [x] CHK008 Специфицированы ли **все пять операций** (install, enable, disable, update, remove) как требования? [Completeness, Покрыто, Spec §FR-015]
- [ ] CHK009 Определён ли **результат каждой операции** над состоянием реестра/конфига однозначно? [Clarity, Двусмысленно, Spec §FR-015 (общая формулировка без детализации по операциям)]
- [ ] CHK010 Заданы ли **граничные случаи** install: плагин уже установлен, недоступный источник, повреждённая загрузка? [Edge Case, Пробел, Gap]
- [ ] CHK011 Заданы ли граничные случаи **enable/disable**: включение несуществующего/уже включённого, отключение зависимости другого плагина? [Edge Case, Пробел, Gap]
- [ ] CHK012 Заданы ли граничные случаи **update**: обновление до несовместимой версии, откат при неуспехе, обновление всех vs одного? [Edge Case, Пробел, Gap]
- [ ] CHK013 Заданы ли граничные случаи **remove**: удаление плагина, от которого зависят другие; удаление включённого плагина? [Edge Case, Пробел, Gap]
- [x] CHK014 Определено ли, что операции управления плагинами выполняются **только средствами CLI** без ручного редактирования артефактов? [Measurability, Покрыто, Spec §SC-009]
