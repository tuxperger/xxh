## Notes

- Статусы отражают spec.md (US7, FR-033..040, SC-010..012) + contracts на 2026-07-03.
- **Отсутствующие / недоопределённые требования (по вашему запросу выделены явно)**:
  - **[Пробел] Nix без флейков** (CHK008) — задан лишь случай «нет Nix»; поведение при
    установленном Nix без поддержки флейков не специфицировано.
  - **[Пробел] Реакция на недоставленные рантайм-данные** (CHK021) — есть тип ошибки в
    контракте, но нормативного требования (что показать/как деградировать) в §FR нет.
  - **[Двусмысленно → к подъёму в §FR]**: критерий самодостаточности бинаря (CHK012),
    критерий «не собирается статически» (CHK027), таблица выбора target (CHK017),
    исчерпывающий перечень классов рантайм-данных (CHK020), клиентское кеширование по
    выходному хешу (CHK033).
- **Системный паттерн spec↔contracts**: значительная часть детального контракта Nix-
  провайдера задана в `contracts/nix-provider.md`, но не поднята в нормативные `§FR`
  spec.md. Рекомендация: поднять критерии (самодостаточность, target-таблица, рантайм-
  данные, клиентский кеш) в spec как FR либо явно сослаться из spec на контракт.
- Пересечения: CHK032 ↔ platform-delivery.md CHK030 и plugins.md CHK030/032 (единство
  очистки/кеша между базой, плагинами и Nix).
- Отмечайте выполненные: `[x]`. Приоритет к `/speckit-clarify`: CHK008 (Nix без флейков),
  CHK021 (недоставленные рантайм-данные), CHK012/CHK027 (критерии статичности).
