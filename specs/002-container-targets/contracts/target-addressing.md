# Contract: адресация целей (CLI и конфиг)

**Crate**: `xxh-cli`, `xxh-config` | **Principles**: III, VII, XI

Одна позиционная «цель» в CLI и одна строка адреса в конфиге описывают и SSH-хосты,
и контейнеры — единообразно, но различимо (схемный префикс). Research R4.

## Грамматика

```text
target        = ssh-target | container-target
ssh-target    = [ "ssh:" ] [ user "@" ] host        ; как в 001, без изменений
container-target = runtime-scheme ":" container-ref
runtime-scheme   = "docker" | "podman" | "container"   ; (+ "containerd"/"k8s" за feature-флагами)
container-ref    = имя или id контейнера, непустой
```

Примеры: `xxh myhost`, `xxh deploy@10.0.0.5`, `xxh docker:app1`,
`xxh podman:6f0a12`, `xxh container:app1 --runtime podman`.

## Обязательства

- **C-A1**: Строка без известной схемы трактуется как SSH-цель — существующие
  вызовы и конфиги работают без правок (обратная совместимость).
- **C-A2**: Неизвестная схема — ошибка конфигурации ДО каких-либо подключений, с
  перечнем поддерживаемых схем; пустой `container-ref` — аналогично.
- **C-A3**: `container:` разрешается в конкретный рантайм детерминированно:
  `--runtime` флаг > per-target конфиг > `container.runtime` > авто-порядок
  docker → podman (первый доступный). Выбранный рантайм печатается в verbose.
- **C-A4**: `docker:`/`podman:` НЕ подменяются друг другом ни при какой
  недоступности (нет тихого fallback; различимая ошибка).
- **C-A5**: Флаги/ключи сессии делятся на два класса (перечень исчерпывающий для
  MVP; новые ключи обязаны объявлять свой класс при добавлении):
  - **общие** (одинаковая семантика для обоих семейств): `shell`, `plugins`,
    режим очистки (`cleanup`/`--keep`), `connect_timeout_s`, verbose-уровни,
    `user` — для SSH это пользователь соединения, для контейнера — пользователь
    exec-сессии (аналог `-u` рантайма);
  - **ssh-only** (для контейнерной цели отклоняются с понятной ошибкой до
    подключения): `identity` (`-i`), `port` (`-p`), `transport`
    (выбор russh/ssh-cli — бэкенды семейства SSH; рантайм контейнеров выбирается
    отдельным ключом `container.runtime`/флагом `--runtime`, который, в свою
    очередь, неприменим к SSH-целям).
- **C-A6**: Ключ `container.runtime` входит в канонический конфиг, JSON-схему и
  Nix-модуль-генератор (Принцип XI); рантайм-зависимости от Nix не появляется.

## Тестируемость

Разбор адресов — чистая функция: таблица unit-тестов (все формы, ошибки схем,
precedence рантайма) без docker/сети.
