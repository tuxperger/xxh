# Contract: Transport Trait v2 (обобщение на семейства)

**Crate**: `xxh-transport` | **Principle**: III (Абстракция транспорта, конституция v1.5.0)

Расширяет contracts/transport-trait.md фичи 001. Trait остаётся ЕДИНЫМ; меняется только
тип цели. Три реализации: `RusshTransport`, `SshCliTransport` (семейство SSH),
`ContainerCliTransport` (семейство container-runtime).

## Trait (эскиз изменений)

    async fn connect(&mut self, target: &ResolvedTarget, auth: &AuthPolicy)
        -> Result<(), TransportError>;

    // exec / upload_stream / open_pty / disconnect — сигнатуры и семантика 001
    // БЕЗ ИЗМЕНЕНИЙ (это и есть суть контракта).

## Обязательства (дополнительно к C-T1..C-T5 из 001)

- **C-T6**: Бэкенд, получивший цель чужого семейства, возвращает
  `BackendUnavailable` немедленно, без сетевых/процессных действий и без тихого
  fallback на другой бэкенд.
- **C-T7**: `Session`, плагины, шеллы и очистка НЕ содержат ветвлений по семейству
  транспорта; единственная точка выбора бэкенда — фабрика в `xxh-cli` по
  `ResolvedTarget` + конфигу.
- **C-T8**: `AuthPolicy` относится только к семейству SSH; контейнерный бэкенд обязан
  игнорировать её и НЕ запрашивать ssh-учётные данные (Принцип V; FR-009 спеки).
- **C-T9**: C-T1 (секреты не логируются) расширяется: пути к сокетам рантайма и
  значения переменных окружения exec-команд не попадают в лог ни на одном уровне.
- **C-T10**: Существующие SSH-вызовы компилируются и ведут себя идентично после
  обобщения типа цели (обратная совместимость поведения; SC-006 спеки).

## Тестируемость

- Все три бэкенда проходят один и тот же интеграционный набор «заход → свой шелл →
  выход → чистота» (contracts/dual-transport-testing.md).
- C-T6/C-T8 покрываются unit-тестами бэкендов без реального рантайма.
