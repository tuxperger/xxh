# Checklist: качество требований к контейнерному транспорту

к транспорту для подключения к контейнерам перед `/speckit-tasks`
**Created**: 2026-07-07
**Feature**: [spec.md](../spec.md) · [plan.md](../plan.md) · [contracts/](../contracts/)

Вердикты: ✅ Покрыто · ⚠️ Двусмысленно · ❌ Пробел

## Обобщение транспортного интерфейса

- [x] CHK001 — Определено ли требование взаимозаменяемости контейнерного бэкенда с
  SSH-бэкендами за единым интерфейсом (connect/exec/доставка/PTY/disconnect)?
  [Completeness, Contract C-T6/C-T10, Конституция III] — **Покрыто**
- [x] CHK002 — Зафиксировано ли, что ядро, плагины, шеллы и очистка не зависят от
  семейства транспорта, и названа ли единственная точка выбора бэкенда?
  [Clarity, Contract C-T7; Spec FR-003] — **Покрыто**
- [x] CHK003 — Специфицировано ли поведение при несоответствии цели и бэкенда
  (без тихого fallback)? [Edge Case, Contract C-T6, C-A4] — **Покрыто**
- [x] CHK004 — Есть ли требование обратной совместимости SSH-пути (существующие
  вызовы/конфиги без правок)? [Consistency, Spec FR-012/SC-006, Contract C-A1] — **Покрыто**

## Адресация цели и выбор рантайма

- [x] CHK007 — Определён ли единый, но различимый способ адресации хостов и
  контейнеров в CLI (схемные префиксы, грамматика, ошибки разбора)?
  [Clarity, Contract target-addressing C-A1/C-A2] — **Покрыто**
- [x] CHK008 — Специфицирован ли детерминированный выбор рантайма при нескольких
  доступных (precedence: флаг > конфиг > авто-порядок; видимость выбора в verbose;
  неоднозначное имя контейнера)? [Completeness, Spec FR-010/US5, Contract C-A3] — **Покрыто**
- [x] CHK009 — Определено ли, какие per-target ключи конфига применимы к контейнерным
  целям, а какие (identity, port) — только к SSH, и полный ли этот перечень?
  [Ambiguity, Contract C-A5] — **Покрыто** (T002): C-A5 дополнен исчерпывающим
  перечнем «общие vs ssh-only»; `user` объявлен общим ключом (для контейнера —
  exec-пользователь), что заодно закрывает источник `exec_user` в data-model.

## Zero-footprint в контейнере

- [x] CHK010 — Зафиксировано ли, что образ и слои не модифицируются (запрет
  commit/build/записи вне tmp), а окружение живёт в одной tmp-директории внутри
  запущенного контейнера? [Completeness, Spec FR-006, Contract C-C12/C-C13,
  Конституция I] — **Покрыто**
- [x] CHK011 — Определены ли правила выбора tmp-директории для непривилегированного
  пользователя и поведение при полном отсутствии записываемых мест (ошибка до
  доставки, без полуразвёрнутого состояния)? [Edge Case, Spec FR-011, Contract C-C12] — **Покрыто**
- [x] CHK012 — Специфицирована ли судьба сохранённого окружения (`--keep`) при
  перезапуске/пересоздании контейнера: переживает ли кеш `restart`, что происходит
  при `recreate`, и как это соотносится с «неизменённые артефакты не перекачиваются»?
  [Gap, Spec Edge «Повторное подключение», Принцип VI] — **Покрыто** (T003): spec
  фиксирует — кеш живёт в ФС контейнера (переживает stop/restart, исчезает при
  recreate), повторный вход валидирует по контент-хешу и sweep-ит устаревшее.

## Платформа и минимальные образы

- [x] CHK013 — Требуется ли определение платформы (ОС/арх/libc) внутри контейнера тем
  же механизмом, что для SSH-хостов, включая случай эмулированной архитектуры?
  [Consistency, Spec FR-005, Contract C-C7] — **Покрыто**
- [x] CHK014 — Назван ли Alpine/musl/BusyBox основным кейсом с обязательным покрытием?
  [Coverage, Spec US2/SC-002, Contract C-DT1] — **Покрыто**
- [x] CHK015 — Согласованы ли spec и plan в определении «минимального гарантированного
  набора возможностей цели»: spec обещает «крайне ограниченный тулинг — окружение всё
  равно разворачивается», plan (R3) требует POSIX `sh` и исключает scratch-образы?
  [Conflict, Spec Edge «Крайне ограниченный тулинг» vs Research R3, Contract C-C6] —
  **Покрыто** (T001): граница поднята в spec (Edge Cases + Assumptions): POSIX `sh`
  обязателен (BusyBox достаточно), scratch/distroless — явная ошибка, вне MVP.
